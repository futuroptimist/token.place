# token.place v0.1.1 release validation record

This page captures the v0.1.1 release guardrails and evidence template. Keep completed command
outputs, artifact digests, desktop build links, and manual validation notes here or in the linked
promotion issue so `docs/PRODUCTION_PROMOTION.md` can stay evergreen.

## Release metadata

- App/desktop release version: `0.1.1`.
- API v1 model: `llama-3.1-8b-instruct`.
- Helm chart package version: `0.1.2` when deploying the current chart package.
- Helm chart `appVersion`: `0.1.1`.
- Chart `version` and `appVersion` may intentionally differ because chart `version` is the immutable
  OCI deployment package version, while `appVersion` is the token.place app release version shown by
  public release metadata.

## Behavior validated for v0.1.1

- Landing page renders a public-safe environment/version badge for production, staging, and local
  development.
- Desktop supports multiple compute-node relay URLs and migrates persisted config from the legacy
  single Relay URL field.
- A single desktop node can register with both `https://token.place` and
  `https://staging.token.place` at the same time for v0.1.x because API v1 exposes one model,
  `llama-3.1-8b-instruct`.
- Multi-relay desktop operation reuses the shared warmed llama.cpp runtime instead of requiring one
  model warm-load per relay.
- Relay URL fields are stopped-only; changes apply on the next operator start.
- Per-relay failures are isolated: one relay may fail while others continue polling, and status shows
  the registered count rather than leaking full public keys.
- Stop attempts to unregister from every configured relay; diagnostics should drop or expire both
  registrations.
- Relay logs and diagnostics remain ciphertext only plus safe routing metadata.

## Manual prod + staging validation evidence template

Record the exact desktop artifact, OS, model path/fingerprint if available, and timestamps for each
step below:

1. Build and install desktop on Windows and macOS.
2. Configure relay URLs:
   - `https://token.place`
   - `https://staging.token.place`
3. Start operator and confirm desktop shows registered `2/2`.
4. Confirm `https://token.place/relay/diagnostics` increments
   `total_api_v1_registered_compute_nodes`.
5. Confirm `https://staging.token.place/relay/diagnostics` increments
   `total_api_v1_registered_compute_nodes`.
6. Send one landing chat through production.
7. Send one landing chat through staging.
8. Stop one relay or one route if practical and confirm the healthy relay continues serving work.
9. Stop operator and confirm both registrations disappear or expire.
10. Confirm relay logs remain relay-blind: no plaintext prompts, responses, tool arguments, model
    output, private keys, decrypted payloads, or full public keys.
